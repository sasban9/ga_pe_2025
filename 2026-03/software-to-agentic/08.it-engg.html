Department Deep Dive · 2026
IT &EngineeringSoftware Stack → AI Agentic Stack · Deep Mapping

The most structurally disrupted function. The shift rewires how software is built, deployed, and maintained — not incrementally, but fundamentally. Engineers stop writing instructions for machines and start designing systems that generate and execute instructions themselves.

Core Transformation · 1 of 2

Software Stack Era — Deterministic

Input → Human writes code → System executes → Output

  • Inputs: Requirements documents, Jira tickets, technical specs
  • Processing: Developers write every line of logic manually
  • Output: Features, APIs, deployed systems
Control = Human-driven execution. Every behaviour is explicitly authored. Nothing runs unless a developer has written it. The system has zero ability to act on its own.
02 / 28
Core Transformation · 2 of 2

AI Agentic Stack — Autonomous

Input → AI interprets intent → Agents build & run workflows → Output

  • Inputs: Goals, constraints, natural language instructions
  • Processing: Multi-agent collaboration — no human in the loop for each step
  • Output: Working, continuously evolving systems
Control = Human-guided autonomy. Humans set direction and constraints. Agents handle execution. The system builds, tests, and deploys itself.
03 / 28
Layer A — Requirement → Specification · Before

How Requirements Used to Work

  • Product manager writes detailed PRDs (Product Requirement Documents)
  • Engineers interpret the specs — often introducing ambiguity and gaps
  • Misunderstandings cause delays, rework, and scope creep
  • Input type: Static documents — written once, rarely updated mid-sprint
The handoff from product to engineering is the biggest single source of wasted time in the software era. Requirements are written in natural language, then re-interpreted in code — two lossy translations before anything is built.
04 / 28
Layer A — Requirement → Specification · After

How Requirements Work Now

  • Human provides intent: "Build a payment API with fraud detection"
  • Spec Agent converts intent → full technical architecture automatically
  • System iterates interactively — asks clarifying questions, refines in real time
  • Input type: Structured intent + constraints — living, dynamic, immediately actionable
Documentation-heavy handoff→Dynamic spec generation

Layer C — Testing & QA · Before

How QA Used to Run

  • QA engineers manually design and write test cases for each feature
  • Run test scenarios against builds — human effort required every cycle
  • Bugs discovered → reported to developers → fixed → retested manually
  • Output lag: High — each cycle takes days; edge cases frequently missed
QA in the software era is reactive by design. Tests are written after code. Bugs are found after users encounter them. The feedback loop is always one step behind.
08 / 28
Layer C — Testing & QA · After

How QA Works Now

  • Test Agent auto-generates test cases as code is written — no separate QA cycle
  • Runs simulations continuously in the background — always-on validation
  • Detects edge cases proactively before users ever encounter them
  • Output: Near real-time validation — feedback in minutes, not days
Reactive QA→Continuous autonomous validation

Layer D — Deployment & DevOps · Before

How Deployments Were Managed

  • CI/CD pipelines configured manually — Jenkins, CircleCI, GitHub Actions
  • Infrastructure set up by hand: AWS consoles, Docker configs, YAML files
  • Engineers monitor every deployment — watching logs, ready to intervene
  • Workflow: Build → Test → Deploy — each stage human-gated, each config handcrafted
DevOps in the software era is a highly skilled manual job. Every config, every rollback, every infra change requires a specialist who understands all moving parts.

New Engineering Stack — Agentic Architecture · Layers 1 & 2

Intent & Agent Layers

1

Intent Layer

The Intent Layer is where human creativity lives. The Agent Layer is where it gets built, tested, shipped, and maintained — automatically.

Workflow & System Layers

3
WORKFLOW
Workflow Layer
Orchestration across all agents. Event-driven execution — one agent's output triggers the next. Cross-agent handoffs happen automatically based on defined conditions.

System Layer
Live, deployed applications running in production. Software that continuously improves itself — updates, patches, optimisations — without a fixed release schedule.
The Workflow Layer is the nervous system connecting all agents. The System Layer is the living product — never static, always evolving.

Skills 01 — 03

01
System Design > Coding
Architecture trumps implementation. Understanding how pieces connect matters more than being able to write each piece individually.
02
Architecture Thinking > Implementation
Designing systems at a structural level — data flows, service boundaries, agent responsibilities — becomes the primary engineering skill.
03
Constraint Definition > Detailed Specs
Instead of 50-page specs, engineers define boundaries: what the system must not do, what resources it can use, what risks are acceptable.
