Chapter 01 — The Old World

What Was an SOP?

Standard Operating Procedures were written for humans — static documents, binder-bound, rarely updated, siloed in departments.

Step 1. Open the spreadsheet.
Step 2. Enter the data manually.
Step 3. Email manager for approval.
Step 4. Wait 3–5 business days.
SOP
Chapter 02 — The Inflection Point

The World Changed.
Procedures Didn't.

AI agents can now read, decide, act, and report — yet most organizations still write procedures as if a human with a checklist is the executor.

The gap between what agents can do and how we instruct them is the biggest operational risk of the decade.

Chapter 02 — Definition

What Is an Agent Protocol?

An Agent Protocol is a structured instruction set designed for autonomous AI systems — encoding not just steps, but context, decision logic, error handling, and escalation paths.

protocol: onboard_new_client
trigger: contract_signed == true
executor: agent_cluster_A
fallback: human_review
success_criteria: 95% confidence

Chapter 05 — Oversight

Humans Are Still In The Loop

Agent Protocols don't eliminate human judgment — they reserve it for the moments it matters most.

  • Routine, high-volume tasks → fully autonomous
  • Ambiguous or high-stakes decisions → human review gate
  • Novel edge cases → protocol flags for escalation
  • Audit & compliance checks → human sign-off required

Chapter 08 — Craft

How to Write an Agent Protocol

  • Be precise: Ambiguity in human SOPs is forgiven; agents execute literally
  • Define success: What constitutes task completion? What metrics confirm it?
  • Map failure modes: Every branch must have a defined error path
  • Version everything: Protocols evolve — track changes like code
  • Test before deploying: Simulate with sandboxed agents first

Chapter 10 — Governance

Compliance Is Now Embedded

Agent Protocols don't just execute — they document. Every action is logged, timestamped, and traceable, making compliance a byproduct of operations, not an afterthought.

Audit trails are automatic. Regulators get a full execution log, not a reconstruction from memory.

Chapter 11 — Risk

What Breaks When Protocols Fail

  • Ambiguous trigger conditions → agent fires at wrong time
  • Missing fallback states → process hangs indefinitely
  • No confidence threshold → low-quality outputs accepted
  • Stale context → agent acts on outdated information
  • No kill switch → runaway agent can't be stopped

Chapter 16 — Continuity

What Doesn't Change

At their core, SOPs and Agent Protocols share the same purpose: consistency, quality, and accountability.

  • The goal of reducing variation and error remains
  • Human values and business ethics must still guide every protocol
  • Training and comprehension — now of the agents, not just people
  • Continuous improvement remains the north star
Chapter 17 — Pitfalls

5 Mistakes to Avoid

  • Copy-pasting SOPs verbatim as agent prompts
  • Assuming the agent understands implicit context humans take for granted
  • Building protocols without a defined success metric
  • Skipping edge-case mapping and treating "happy path" as the only path
  • Deploying at scale before human-supervised pilot testing
