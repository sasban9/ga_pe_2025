Definition

What Is NLP-as-Code?

  • Using natural language prompts as the primary unit of software construction
  • LLMs act as compilers that translate intent → executable code
  • Iteration happens in conversation, not in an IDE
  • The "program" is the prompt stack + generated artifacts
  • Testing = describing expected behavior in plain language
Before — Traditional Code

The Old Ritual

# Sort users by signup date, then email 30-day inactives import datetime from db import get_users, send_email users = get_users() sorted_users = sorted(users, key=lambda u: u.signup) threshold = datetime.datetime.now() - datetime.timedelta(days=30) for u in sorted_users: if u.last_active < threshold: send_email(u, "We miss you!")

Hours of writing, testing, and debugging — for a task you described in one sentence.

After — NLP-as-Code

The New Ritual

PROMPT → Get all users from the database sorted by signup date. Find anyone inactive for 30+ days and send them a "We miss you!" re-engagement email. OUTPUT → Working Python code generated instantly. Tests, edge cases, error handling included on request.█

The intent is the code. Language is the interface.

The Headline Claim

Python is now what
Assembly was in 1975.

Assembly didn't disappear — but only specialists write it. Python is becoming the low-level target that LLMs emit. Humans define outcomes; machines produce syntax.

Nobody writes Assembly to build a web app.
Soon, nobody will write Python to build one either.

Prompt Engineering
IS Engineering

# A well-engineered prompt is a specification: CONTEXT: SaaS billing system, Stripe integration TASK: Handle failed payments with exponential retry RULES: Max 3 retries, notify user on 2nd fail OUTPUT: Python service + unit tests + error logs AVOID: Polling loops; use webhooks█

Balance

Python Doesn't Die — It Descends

  • ML researchers hand-craft training loops, custom kernels
  • Security engineers audit generated code at the Python level
  • Python becomes the inspection layer, not the composition layer
  • Just as C is still used — but not by most developers, most of the time

The Other Side

Risks & Critiques

  • Hallucinated code — LLMs confidently write plausible-but-broken logic
  • Security surface — generated code often misses auth, injection, race conditions
  • Skill atrophy — developers who can't read generated code can't fix it
  • Prompt lock-in — proprietary prompt stacks become unmaintainable
  • Non-determinism — same prompt, different behavior next week
Critical Skill

Verification
is the New Coding

If you can't read what the LLM wrote, you can't ship it safely. Code literacy remains essential — just not code authorship.

In Practice

A Real NLP-as-Code Pipeline

1. SPEC → Write intent in plain English (Notion, doc) 2. GEN → LLM generates Python + tests + docs 3. REVIEW → Human reads & audits generated code 4. TEST → CI runs generated test suite 5. SHIP → Deploy; LLM monitors logs, suggests fixes 6. ITER → New behavior? Update the prompt spec.█

The prompt spec becomes the source of truth. Python becomes an artifact.
