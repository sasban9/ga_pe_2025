The Death of an Era · 2025

PROMPT
ENGINEERING

"Context Design" is the new high-ground.

The Provocation

What if the skill was never about writing clever sentences to a model — but about designing the world the model wakes up inside?

First, let's be honest

What WAS
Prompt Engineering?

Carefully crafted one-shot instructions sent to a model. Magic phrases. Chain-of-thought tricks. Role-playing gambits. A bag of micro-techniques for extracting better outputs from a static, stateless text window.

But then — the world changed

  • 01 Models gained 200K+ token context windows
  • 02 Agents started multi-step reasoning across tools
  • 03 RAG, memory & tool-use became standard
  • 04 Systems replaced single turns with long-running loops
  • 05 The prompt became one layer in a deep stack

R.I.P.

PROMPT
ENGINEERING

Not dead because models got dumber.
Dead because they got so smart the bottleneck moved upstream.

Where the bottleneck lives now

The limit is no longer
how you phrase the question

It's what information, structure, and state the model has access to before it begins reasoning. The bottleneck is the context.

The Successor

CONTEXT
DESIGN

The deliberate, architectural practice of shaping what a model knows, believes, and can access at the moment it acts — across every layer of a system.

Old paradigm vs New paradigm

Prompt Engineering

Tweak the instruction. Find the magic phrase. Iterate on wording. Hope the output improves.

VS

Context Design

Architect the information environment. Curate memory. Structure retrieval. Define the world the model inhabits.

The five layers of context

Context isn't a prompt.
It's an architecture.

  • ①System Context — Identity, rules, persona
  • ②Memory Context — Persistent state, past events
  • ③Retrieved Context — Relevant docs, data, tools
  • ④Conversation Context — Turn history, user goals
  • ⑤World Context — Time, location, env signals

An analogy to make it stick

A film director doesn't tell an actor how to cry. They build the world, cast the right people, write the scene — and then the performance emerges.

Prompt engineering = directing by screaming louder.
Context design = production design mastery.

The scale shift

200K

tokens of context

That's ~150,000 words. An entire novel. Three PhD theses. You don't prompt into a context that large. You design it.

Example in the wild

RAG is Context Design

Retrieval-Augmented Generation isn't a prompting trick. It's the practice of dynamically engineering what the model knows before it answers. The retrieval pipeline IS the design.

Example in the wild

System prompts aren't
"just configuration"

// Bad: prompt engineering mindset
system: "Be helpful and accurate."

// Good: context design mindset

        [user profile] [current goals]
        [available tools] [output schema]

What context designers actually do

  • 01Design information hierarchies — what's in vs. out
  • 02Architect memory schemas — what persists, what fades
  • 03Engineer retrieval relevance — chunking, embedding, ranking
  • 04Curate few-shot exemplars as behavioral anchors
  • 05Define tool affordances — what actions are possible

The hidden superpower

Information
Compression

Context windows are finite. The art of context design is knowing what to include, what to summarize, and what to discard — so the model's reasoning window is always filled with signal, never noise.

The anti-pattern to avoid

"Context stuffing"
is not Context Design

Dumping every document, every turn, every instruction into the window is lazy. Context design is curation — surgical, intentional, always asking: does this token earn its place?

The competitive edge

In 2025, the winner is
whoever controls the context.

Two companies. Same model. Same API. Radically different results. The difference is never the prompt. It's the context infrastructure around the model.

The gap to close

We have great tools for prompting.
We barely have tools for context design.

No standard context schema. No shared memory protocols (yet). No "context debugger" in widespread use. The infrastructure is years behind the opportunity.

First principles of context design

  • IRelevance over volume — less, but perfectly chosen
  • IIFreshness matters — stale context poisons reasoning
  • IIIStructure signals intent — formatted context guides action
  • IVThe model is your collaborator, not your servant
